🎨 Design Decisions

Decision Gold + Navy color palette

app.config.ts defines primary: 'gold' and gray: 'navy'. The spec did not mandate colors, but gold/navy are conventional for high-authority legal brands — they signal prestige, trustworthiness, and formality without being cold.

Decision Catch-all routing ([...slug].vue)

Rather than creating individual .vue page files for every service/city/blog route, a single pages/[...slug].vue catches all content-driven URLs and delegates rendering to Nuxt Content. This keeps the page surface minimal and ensures new Markdown files are automatically routable without code changes.

Decision Spanish as default locale (no /es/ prefix)

strategy: 'prefix_except_default' means Spanish URLs are clean (/contacto) while English adds /en/. The target audience is primarily Spanish-speaking; clean URLs also carry minor SEO weight. The English version is present for bilingual SEO coverage.

Decision Inter + Cormorant Garamond type pairing

Inter (sans-serif) handles UI text for readability; Cormorant Garamond (serif) provides a refined, authoritative editorial voice for headings and hero copy — standard in premium legal and consulting websites.

Decision Nuxt 4 with compatibilityVersion: 5

The project targets the Nuxt 4 app directory convention while enabling the v5 compatibility flag to opt into upcoming breaking changes early. This means the project uses app/ rather than the root-level pages/, components/, etc.

↪️ Deviations from Spec

Deviation README was describing a different project

The original README.md contained content from a Mendoza Roofing project — a different client entirely. It was completely rewritten to accurately describe the Karina Orocio Cruz law firm website, its tech stack, services, and project structure.

Deviation nuxt-vitalizer config commented out

nuxt.config.ts has the vitalizer block commented out (disableStylesheets: 'entry'). It was added to fix render-blocking CSS flagged by a Lighthouse audit but was later disabled — likely because it caused hydration or visual issues on the entry stylesheet. Needs re-evaluation before final production deploy.

Deviation routeRules for OG type are commented out

Per-route ogType overrides for /blog/** and /tools/** are commented out in nuxt.config.ts. These were intended but not activated, meaning all routes share the default OG type. Should be re-enabled if blog article sharing is a priority.

Deviation workTranslation on SchemaOrg identity

The schemaOrg.identity block uses a custom workTranslation field which is not standard Schema.org vocabulary. It was probably intended as workExample or a custom extension. This may not be indexed by search engines and should be validated against Google's Rich Results Test.

Deviation Contact phone numbers mismatch

nuxt.config.ts lists only one phone (+52-951-615-30-10), while plan/Company-info.md lists three numbers (9516153010 · 9516117710 · 9516130039). The schema and contact components may not expose all numbers.

⚖️ Tradeoffs

Tradeoff Static generation vs. SSR

The build script is nuxt generate (full static). This means:

  • ✅ Zero cold-start latency, free Netlify hosting tier, easy CDN caching
  • ⚠️ Contact form submissions require an external service (Netlify Forms or third-party API)
  • ⚠️ Content edits require a rebuild + deploy cycle

Tradeoff Catch-all page vs. typed page components

Single [...slug].vue keeps the codebase lean and routes all content without maintenance overhead. The downside is reduced per-route control: custom layouts, breadcrumbs, or component injection per content type must be handled via Markdown frontmatter and conditional component selection inside the catch-all.

Tradeoff pnpm over npm/yarn

pnpm provides faster installs and strict dependency isolation, but requires the ignoredBuiltDependencies workaround for native modules (sharp, esbuild, better-sqlite3). The pnpm-lock.yaml (594 KB) must be committed.

Tradeoff Nuxt Content v3 Markdown files vs. CMS

Content is managed as .md files in the repo. This is ideal for a developer workflow and zero additional cost, but requires Git access to update content. A headless CMS (Storyblok, Sanity) was not introduced to keep the stack simple and avoid ongoing costs for a single-attorney practice.
